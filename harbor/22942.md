# Braindump for https://github.com/goharbor/harbor/issues/22942

so photon is just a lightweight os, we are using photon base image.

`$(DOCKERBUILD) --pull --no-cache -f Dockerfile.base ...`

here we are pulling the lastest image first and then building. and the FROM line has photon:5.0 .

So we are pulling latest photon:5.0 image , instead of using a locally cached one. problem with that is if any vul fix is added to that photon package repos but that photon image itself is not rebuilt and pushed - then harbor will not have that vul fix.

what we should do:

we should use tdnf to update the packages ourselves. I see that harbor has this dockerfile.base that is the base image and that is used in actualy dockerfile of that component. so we will update all the dockerfile.base with this `tdnf upgrade -y`
